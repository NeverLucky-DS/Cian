# change 2: web-просмотрщик БД

## что добавлено

`viewer.py` — маленький Flask-сервер, чтобы человек мог глазами смотреть, что лежит в БД, не лазая в SQL.

### функции

- `index()` (роут `/`)
  - список объявлений плиткой (адаптивная сетка), пагинация по 30 шт.
  - фильтры в URL: `q` (поиск по описанию/адресу/ЖК/заголовку через ILIKE), `rooms`, `price_max`, `nb` (только новостройки/только вторичка), `sort` (новые/цена ↑/цена ↓/площадь ↑).
  - на каждой карточке цена, площадь, этаж, адрес, метро, ЖК, обложка (первое скачанное фото из `offer_photos`).
- `detail(cian_id)` (роут `/offer/<cian_id>`)
  - детальная карточка: вся «плоская» инфа из `offers` справа, галерея слева с lightbox по клику, описание, ссылка на оригинал на Cian, телефоны продавца.
- `photo(cian_id, fname)` (роут `/photos/<cian_id>/<position>.webp`)
  - отдает webp-файлы из локальной папки `photos/`.

### зависимости
- добавлен `flask>=3.0` в `pyproject.toml`.

### как запускать

```
uv sync
uv run python viewer.py
# открыть http://127.0.0.1:5005
```

Это самый простой инструмент для визуальной проверки качества парсинга. SQL-консоль (`docker exec -it cian-pg psql -U cian -d cian`) остается для глубоких запросов.

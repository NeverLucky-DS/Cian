# change 4: фикс превью и snapshot.py

## превью обложек на главной

Был баг: на главной у всех 56 карточек было «нет фото», хотя 225/226 файлов скачались.

Причина: в `viewer.py` стояло `{% if o.cover %}`, где `o.cover` — это число (позиция фото). Когда позиция = 0, Jinja считает её ложной и рендерит ветку «нет фото». Поправлено на `{% if o.cover is not none %}`.

Заодно — у Cian первое фото (`position=0`) часто это **планировка** (`is_layout=True`), а не сама квартира. Поэтому в `index()` теперь сначала ищем минимальную позицию **где `is_layout=false`**, и только если не нашли — берём любое скачанное фото. Так на превью попадает «настоящая» фотография комнаты (как правило `1.webp`).

## snapshot.py

Падало `git add data/.gitignore` потому что DVC иногда не создаёт этот файл (если артефакт уже покрыт корневым `.gitignore`). Заменил на `git add data` — git рекурсивно подхватит то, что есть, или ничего.

## про ошибку 403 при скачке одного фото

`[photos] 75/226 ERROR id=1028: 403 Forbidden ... 2864229165-1.jpg` — у одного объявления Cian/CDN отдал 403 на конкретный URL. Это не наш баг (после 3 ретраев всё равно 403), просто фото удалено или ограничено на стороне Cian. На остальные 225 не повлияло. Если очень нужно — можно перепарсить эту карточку (`offers --limit 1` через ручной фильтр), но 99 из 100 случаев это не имеет смысла.
